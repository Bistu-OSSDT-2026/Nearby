## 如何报告Bug

如果你发现了Bug，请通过 [Issues](https://github.com/Bistu-OSSDT-2026/Nearby/issues) 提交，并遵循以下模板：

**标题格式**：`[Bug] 简短描述问题`

**内容模板**：

### 问题描述
（清晰描述遇到的问题）

### 复现步骤
1. 访问 xxx 页面
2. 点击 xxx 按钮
3. 出现 xxx 错误

### 预期行为
（描述正常情况下应该发生什么）

### 截图/日志
（如有，请附上相关截图或错误日志）

### 环境信息
- 操作系统：Windows/Mac/Linux
- 浏览器及版本：Chrome xxx

## 开发环境搭建

1. **Fork 本项目** 到你的个人仓库

2. **克隆你的 Fork** 到本地

   git clone https://github.com/你的用户名/Nearby.git
   cd Nearby

3. **添加上游仓库**（便于同步主仓库更新）

   git remote add upstream https://github.com/Bistu-OSSDT-2026/Nearby.git

4. **安装依赖并启动项目**

   cd server
   npm install
   npm start

## 代码提交规范

提交信息（Commit Message）请遵循以下格式：

<类型>(<范围>): <简短描述>

<详细描述（可选）>

**类型说明**：

| 类型 | 说明 |
|------|------|
| feat | 新增功能 |
| fix | 修复Bug |
| docs | 文档更新 |
| style | 代码格式调整（不影响功能） |
| refactor | 代码重构 |
| test | 测试相关 |
| chore | 构建工具、依赖等杂项 |

**示例**：

feat(商品): 增加按价格区间筛选功能

实现了前端价格区间选择组件和后端查询接口

## Pull Request 流程

1. **确保代码已更新**：提交PR前，请先同步主仓库最新代码

   git pull upstream main

2. **推送到你的Fork仓库**

   git push origin 你的分支名

3. **创建Pull Request**：前往主仓库点击 "New Pull Request"

4. **填写PR描述**：
   - 本次改动解决了什么问题
   - 改动涉及哪些模块
   - 是否已本地测试通过

5. **等待Code Review**：至少需要一位项目成员审核通过后方可合并

6. **及时响应反馈**：根据评审意见修改代码并更新PR
